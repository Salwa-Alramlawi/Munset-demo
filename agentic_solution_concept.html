Agentic Solution Concept

Munset — Multi-Agent Judicial Session Documentation System
Track 04: Agent-to-Agent (A2A)

1 What the Agent System Does

Munset is a multi-agent system that automates the documentation and analysis of court sessions in Saudi courts. A judge or court clerk uploads an audio recording of a court session, and the system produces:

  • Structured transcript with speaker identification (judge, plaintiff, defendant, lawyers)
  • Legal analysis — automatically extracted claims, defenses, evidence, legal articles, and contradictions
  • Official session minutes — structured court record + executive summary + next actions
  • Quality review — completeness score, missing information, accuracy verification
  • Interactive chatbot — the judge can ask any question about the session and get an instant answer
  • Cross-session contradiction detection — compares statements across multiple sessions of the same case
Core Value Proposition: Saves 70-80% of documentation time, eliminates human transcription errors, provides instant legal analysis, and enables contradiction detection that would be impractical manually — especially in complex, multi-session cases.

3 System Type: Multi-Agent with Bidirectional A2A

Munset is a multi-agent system with 5 specialized agents communicating via a custom Agent-to-Agent (A2A) protocol.

Why Multi-Agent (Not Single-Agent)?

  • Separation of concerns: Each agent has a specialized prompt, model configuration, and domain expertise. A single agent trying to do transcription + legal analysis + summarization + QA would produce lower quality results.
  • Bidirectional communication: Agents can challenge and improve each other's work — the QA agent sends feedback to upstream agents, creating a self-improving loop.
  • Composability: Each agent can be upgraded independently (e.g., swap Whisper for a better Arabic ASR model without touching the legal analysis agent).
  • Transparency: The A2A message log provides a full audit trail of how each decision was made and which agent made it — critical for a judicial system.

The 5 Agents

🎙️ Transcription Agent LOCAL ML

Tools: Whisper large-v3 (ASR), pyannote 3.1 (speaker diarization), ffmpeg (audio conversion)

Output: Timestamped transcript with speaker IDs mapped to roles

⚖️ Legal Analysis Agent CLAUDE API

Tools: Claude API with specialized Saudi legal system prompt

Output: Structured JSON — case type, claims, defenses, evidence, articles, contradictions, timeline

📝 Summary Agent CLAUDE API

Tools: Claude API with court-clerk persona prompt

Output: Official session minutes + executive summary + next actions list

✅ QA Agent CLAUDE API

Tools: Claude API + A2A bidirectional messaging (sends feedback & clarification requests)

Output: Completeness score, issues list, verified articles, feedback to upstream agents

💬 Chatbot Agent CLAUDE API

Tools: Claude API with full session context + conversation memory

Output: Natural-language answers to judge's questions, with conversation history for follow-ups

What Makes This A2A (Not Just a Pipeline)?

Most multi-agent systems pass data in one direction (Agent 1 → 2 → 3 → 4). Munset's A2A protocol supports bidirectional communication:

  • Forward flow (data): Transcription → Legal Analysis → Summary → QA → Chatbot
  • Backward flow (feedback): QA Agent → Summary Agent (quality notes)
  • Backward flow (clarification): QA Agent → Legal Analysis Agent (questions about legal interpretation)
  • Message tracing: Every message has a unique ID and parent_id, enabling full conversation chain reconstruction
  • Priority system: Clarification requests are marked as high priority
  • Statistics: Per-agent message counts (sent, received, clarifications, feedbacks) visible in the UI

A2A Protocol Message Types

Message Type Direction Description Example
data Forward Primary data flow between sequential agents Transcription Agent sends transcript to Legal Analysis Agent
response Return Agent replies to a data request Legal Analysis Agent returns structured JSON
feedback Backward QA Agent sends quality improvement notes to upstream agents "Missing: name of the engineering office that prepared the plaintiff's report"
clarification_request Backward Downstream agent asks upstream agent for legal clarification "Does Article 79 of the Civil Transactions Act apply to private construction contracts?"
clarification_response Forward Upstream agent answers the clarification "Yes, Article 79 applies — force majeure is a general principle not limited to government contracts."

Memory & Persistence

The system includes a Session Store (file-based JSON database) that persists each processed session with its full analysis. This enables:

  • Case history: Browse all sessions for a given case number
  • Cross-session contradiction detection: Compare a speaker's statements across sessions to detect inconsistencies
  • Per-speaker statement extraction: Tracks what each party said in every session
