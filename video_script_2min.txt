===================================================

المدة: دقيقتان
النصيحة: سجّل الشاشة وأنت تشرح + اعرض الديمو


──────────────────────────────────────
[0:00 - 0:20] المقدمة والمشكلة
──────────────────────────────────────

"السلام عليكم، أنا [اسمك] وهذا مشروع مُنصِت.

في المحاكم السعودية، توثيق الجلسات القضائية يتم يدوياً بواسطة كاتب الضبط.
المشكلة؟ تفاصيل مهمة تضيع بسبب سرعة الحديث، القاضي يقضي وقت طويل في مراجعة المحاضر، وكشف التناقضات في أقوال الأطراف عبر عدة جلسات شبه مستحيل يدوياً."


──────────────────────────────────────
[0:20 - 0:50] الحل
──────────────────────────────────────

"مُنصِت حل هذي المشكلة بمنظومة 5 وكلاء ذكيين يتواصلون عبر بروتوكول Agent-to-Agent.

المستخدم يرفع تسجيل صوتي للجلسة، وخلال دقائق يحصل على:
- نص مُفرّغ مع تحديد كل متحدث
- تحليل قانوني كامل: ادعاءات، دفوع، أدلة، تناقضات
- محضر جلسة رسمي مُهيكل
- تقرير جودة يراجع المحضر تلقائياً
- ومساعد تفاعلي يقدر القاضي يسأله أي سؤال عن الجلسة"


──────────────────────────────────────
[0:50 - 1:20] الميزة التقنية — A2A
──────────────────────────────────────

"اللي يميّز مُنصِت هو بروتوكول التواصل الثنائي بين الوكلاء.

مو بس pipeline بإتجاه واحد — وكيل المراجعة يقدر يرجع ويرسل ملاحظات جودة لوكيل التلخيص، ويطلب توضيحات من وكيل التحليل القانوني. هذا تواصل حقيقي بين الوكلاء، مو مجرد تسلسل.

كل رسالة بين الوكلاء مُسجّلة ومُتتبّعة — وهذا مهم جداً في بيئة قضائية تحتاج شفافية كاملة.

النظام يشتغل بالكامل عبر Cloud APIs — بدون أي GPU أو موديل محلي."

===================================================
  ملاحظات التسجيل
===================================================

1. سجّل بالجوال أو بالشاشة (screen recording)
2. الديمو يشتغل بدون API — شغّله من الموقع أو محلياً
3. خلّ صوتك واضح وهادئ
4. لو تبي تختصر: المقدمة + الحل + الديمو = أهم ثلاث أجزاء
